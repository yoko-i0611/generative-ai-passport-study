# コンテンツ作成フォーマット（第1章基準）

## 基本方針

各ユニットのコンテンツは、以下の2つの要素で構成されます：

1. **textContent（テキスト内容）**
2. **importantExplanations（重要ポイントまとめ）**

## 1. textContent（テキスト内容）

### 目的
公式テキストとシラバスから押さえておく点を**広く簡潔に**記載します。

### 記載内容
- 公式テキストと最新シラバスに照らし合わせた時に重要となる部分を簡潔に広く記載
- 初学者向けの表現で、1つ1つの文章は簡潔に
- 専門用語は最小限にし、わかりやすい言葉に置き換え
- **公式テキストの丸写しは避ける**（公式テキストの内容を理解した上で、独自の表現で書き直す）

### 特徴
- **広範囲をカバー**：そのユニットで理解すべき概念を広く網羅
- **簡潔**：長い文章は複数の短文に分割
- **初学者向け**：専門用語を避け、分かりやすい表現を使用

### 例（第1章 Unit 01）
```
AI（人工知能）とは、人間の知能を再現するコンピュータの能力のことです。知覚、認識、理解、学習、問題解決などの能力が求められます。現在のAIは限定的な能力しか持っておらず、人間の知能を完全に再現することはできません。

AIとロボットは別物です。AIは「脳（ソフトウェア）」で、ロボットは「体（ハードウェア）」です。AIは物理的な身体がなくても機能します。

AIの進化は4つのレベルに分けられます。レベル1は制御で、エアコンや冷蔵庫のように決められたルール通りに動きます。レベル2はルールベースで、将棋プログラムのように人間が入力したルールに基づいて動作します。レベル3は機械学習で、検索エンジンのようにデータからルールを学びます。レベル4はディープラーニングで、自動運転のように特徴量も含めて自律的に学習します。
```

## 2. importantExplanations（重要ポイントまとめ）

### 目的
textContentの中でも**より理解が必要なポイントやその理解方法**を記載します。

### 記載内容
- textContentの中から重要なポイントをピックアップ
- 初学者でも分かりやすい表現で、どのように理解すればいいかを深掘り
- カテゴリごとに分けて、理解しやすく整理

### 特徴
- **詳細な説明**：textContentよりも深い理解を促す内容
- **理解の方法**：「要するに」「なぜなら」などの表現で理解を補助
- **カテゴリ分け**：関連する内容をグループ化

### 例（第1章 Unit 01）
```
{
  category: 'AIの定義とロボットとの違い',
  explanation: 'AI（Artificial Intelligence）は「人間の知能を再現する能力」を持つソフトウェアです。「AI＝脳（ソフト）」「ロボット＝体（ハード）」という区別が重要です。AIは物理的な身体がなくても成立します。',
},
{
  category: 'AIの4つのレベル',
  explanation: 'レベル1（制御）：エアコンや冷蔵庫など、決められたルール通りに動く単純制御。レベル2（ルールベース）：将棋プログラムや掃除ロボットなど、多くのルール（知識）を人間が入力したもの。レベル3（機械学習）：検索エンジンなど、データからルールを学ぶが、着眼点（特徴量）は人間が教えるもの。レベル4（ディープラーニング）：自動運転や画像認識など、「特徴量」も含めて自律的に学習するもの。',
},
```

## 3. unitPoint（ユニットのポイント）

ユニット全体の目的を簡潔に記載します。

### 例
```
unitPoint: 'AIとは何かを正確に定義し、「AIの4つのレベル」による分類、現在のAIがどの段階にあるのか（ANI／AGI）、さらに過去3回のAIブームと「冬の時代」の因果関係を理解することが目的です。',
```

## 4. learningTips（学習のコツ）

学習を進める上でのポイントを箇条書きで記載します。改行区切りで複数行に分けて記述します。

### 例
```
learningTips: `AIは「できること」ではなく「どういう仕組みで知能を実現しているか」という視点で整理すると混乱しません

3回のブームは「何が期待され」「なぜ失望されたのか」をセットで覚えると理解が定着します

「レベル1〜4」の具体例（家電、将棋、検索エンジン、自動運転）をセットで覚えましょう`,
```

## 重要な原則

1. **textContent ≠ importantExplanations**
   - textContent: 広く簡潔に、公式テキストの重要点を網羅
   - importantExplanations: その中でもより理解が必要なポイントを深掘り

2. **重複を避ける**
   - textContentとimportantExplanationsは役割が異なる
   - textContentは広範囲、importantExplanationsは深掘り

3. **初学者向け**
   - 専門用語は最小限
   - わかりやすい表現
   - 1つ1つの文章は簡潔に

4. **公式テキストの丸写しを避ける**
   - 公式テキストの文面をそのまま使用しない
   - 公式テキストの内容を理解した上で、初学者向けに簡潔にまとめて記載する
   - 独自の表現と構成を使用
   - 読みやすさを重視
   - 専門用語の定義も公式テキストの文面そのままではなく、わかりやすい表現に置き換える

## textContent作成時のチェックポイント

1. **公式テキストの丸写しがないか確認**
   - 公式テキストの定義文や説明文をそのまま使用していないか
   - 専門用語の説明も独自の表現になっているか

2. **初学者向けの表現になっているか**
   - 専門用語は最小限か
   - わかりやすい言葉に置き換えられているか
   - 1つ1つの文章は簡潔か（長文は複数の短文に分割）

3. **広範囲をカバーしているか**
   - そのユニットで理解すべき概念を広く網羅しているか
   - 最新シラバスの範囲をカバーしているか
